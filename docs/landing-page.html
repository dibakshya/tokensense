Features

Everything you need to control AI costs

Built for developers and teams who want real visibility into AI spending without sending data to anyone.

🔀

Local HTTPS Proxy

Transparently intercepts AI API calls on 127.0.0.1:7890. Works with any tool that respects HTTPS_PROXY — no code changes needed.

🧠

Task Classification

Rule-based engine classifies every request (code generation, debugging, testing, docs, reasoning). Gemini API fallback for ambiguous prompts.

📊

Daily Reports

Terminal and HTML reports with cost breakdowns by task type, tool, and model. Specific savings recommendations — not generic advice.

💡

Model Advisor

Run tokensense ask "..." on any task description to instantly get ranked model recommendations with cost and quality scores.

👥

Team Reports

Export anonymized usage data as JSON. Merge multiple team exports into a single HTML report. Privacy-preserving — no prompt content included.

🔒

Privacy First

Prompt content is read in-memory for classification and immediately discarded. Only metadata (provider, model, token count, cost) is stored. No telemetry ever.

How It Works

Simple setup. Zero cloud dependency.

Tokensense sits between your AI tools and the APIs. It sees what the model sees — and nothing else.

1

Install & Setup

Run the interactive wizard. It generates a local CA certificate, injects it into your OS trust store, sets the HTTPS_PROXY environment variable, and registers the proxy as a background service.

2

Use AI Normally

Your tools (Cursor, VS Code, Claude Desktop, your own scripts) route through the proxy automatically. For supported tools, the proxy does TLS interception. For cert-pinned tools, it runs in metadata-only mode.

3

Classify & Store

Each request's prompt text is read in-memory, classified by task type, then immediately discarded. Only metadata (provider, model, task type, token count, cost) is written to a local SQLite database.

4

Generate Reports

Run tokensense report to see today's breakdown — costs by task type, savings potential, and specific model swap recommendations. Use --html --open for a full visual report in your browser.

5

Ask the Advisor

Run tokensense ask "write unit tests for my auth module" to instantly see which model gives the best value for that specific task — ranked by cost, speed, and quality score.

Installation

Up and running in under 2 minutes

One command to install, one command to set up.

macOS / Linux

curl -fsSL https://raw.githubusercontent.com/dibakshya/tokensense/main/scripts/install.sh | sh
# Then run setup
tokensense setup

Homebrew

brew install tokensense/tap/tokensense
tokensense setup

Windows (PowerShell)

# Run in PowerShell as Administrator
irm https://raw.githubusercontent.com/dibakshya/tokensense/main/scripts/install.ps1 | iex
tokensense setup

Go install

go install github.com/dibakshya/tokensense@latest
# Add Go's bin folder to PATH (once — makes tokensense findable)
echo 'export PATH="$PATH:$HOME/go/bin"' >> ~/.zshrc && source ~/.zshrc
tokensense setup

If you use bash, replace ~/.zshrc with ~/.bashrc

🗑️ Fully uninstalling Tokensense

Run tokensense uninstall first — it removes the service, CA certificate, proxy env vars, and all data in ~/.tokensense/. Then delete the binary (the command can't remove itself while running):

# Step 1 — run this first
tokensense uninstall

# Step 2 — delete the binary
sudo rm /usr/local/bin/tokensense   # installed via curl / install.sh
rm ~/go/bin/tokensense              # installed via go install

# Step 3 — restart your terminal

Privacy

Your prompts are never stored.

We built Tokensense around a simple principle: the tool that helps you optimize your AI usage should never see what you're working on.

🧠

In-Memory Only

Prompt content is read in-memory for classification, then immediately discarded and eligible for garbage collection.

🗄️

Metadata Only in DB

SQLite stores only: provider, model, task type, token count, cost, latency. Never prompt text.

🌐

No External Calls

The proxy runs entirely offline. The only external call is the optional model matrix update from GitHub.

📵

Zero Telemetry

No analytics. No error reporting. No install ID pings. Your usage data belongs to you.

🔑

Local CA Key

The TLS interception CA key is unique per install, stored with 0600 permissions, never transmitted.

🛡️

Metadata-Only Mode

Enable metadata-only mode to skip content reading entirely — for maximum privacy with cert-pinned tools.

For Developers & Agent Builders

Integrate Tokensense into Any AI Tool or Agent

Expose live cost and classification data to your own apps, scripts, or LLM agents via a local JSON API — no cloud, no keys.

🔌 Local JSON API

Run tokensense api to start a lightweight HTTP server on localhost:7891. Any app, agent, or script can query it.

GET
/v1/status

Proxy on/off + today's call count

GET
/v1/report?date=YYYY-MM-DD

Full cost & savings breakdown as JSON

POST
/v1/classify

Classify a prompt → task type + model recommendations

GET
/v1/usage?limit=N

Raw usage records for any date

🐍 Python Agent Example

agent_cost_guard.py
import requests

# Before calling an expensive model, check spend
report = requests.get(
  "http://localhost:7891/v1/report"
).json()

if report["total_cost_usd"] > 0.50:
  print("Daily budget hit — routing to cheaper model")

# Classify a task before picking a model
result = requests.post(
  "http://localhost:7891/v1/classify",
  json={"prompt": "write unit tests"}
).json()

print(result["recommendations"][0]["display_name"])
# → "Gemini 2.5 Flash"
